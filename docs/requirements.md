## 1. プロジェクト概要

### アプリ名

**Ortho Luxmeter**（仮）

### 目的

睡眠の質改善を支援するスマートフォン向け照度計アプリ。起床時の日光浴促進・日中の光量確認・夜間の光量抑制の3つの場面で、ユーザーの光環境を可視化しアドバイスを提供する。

### 対応プラットフォーム

| プラットフォーム | 対応 |
|---|---|
| iOS | 対応 |
| Android | 対応 |

### 表示言語

日本語のみ

---

## 2. 対象ユーザー

- 睡眠改善に関心がある一般ユーザー全般
- 特定の年齢層・職業を問わない

## 3. 機能要件

### 3-1. 照度計測機能

アプリの中核機能。起動と同時にリアルタイム計測を開始する。

| 項目 | 仕様 |
|---|---|
| 計測開始タイミング | アプリ起動と同時に自動開始 |
| 表示単位 | lux |
| 測定上限値 | 300,000 lux（上限を超えた場合は上限値として表示） |
| 表示更新 | リアルタイム（連続更新） |
| 注意表示 | 「あくまで参考値であり正確な値ではない」旨を常時表示 |

#### プラットフォーム別の計測方法

| プラットフォーム | 実装方法 |
|---|---|
| iOS | AVFoundation を使用し、カメラのISO値・シャッタースピードから lux を算出。外部APIは使用しない |
| Android | SensorManager の TYPE_LIGHT（照度センサー）を使用 |

---

### 3-2. 時間帯別アドバイス表示

端末の現在時刻をもとに時間帯を自動判定し、静的テキストでアドバイスを表示する。

#### 時間帯の定義

| 時間帯 | 時刻範囲 | 推奨 lux |
|---|---|---|
| 朝 | 起床〜11時台 | 2,500 lux 以上 |
| 昼 | 12時〜17時台 | 目標 10,000 lux 以上 |
| 夜 | 18時〜就寝前 | 50 lux 以下 |

#### アドバイス表示仕様

- 各時間帯に対応する静的アドバイステキストを表示する
- 現在の測定値と基準値を照らし合わせ、フィードバックテキストを動的に切り替えて表示する
- フィードバック例：
  - 朝・基準未達（2,500 lux 未満）：「もう少し明るい場所に移動しましょう」
  - 朝・基準達成（2,500 lux 以上）：「十分な光を浴びています」
  - 昼・基準達成（10,000 lux 以上）：「沢山浴びてセロトニンを分泌させましょう」
  - 夜・基準超過（50 lux 超）：「光を抑えて寝る準備をしましょう」
  - 夜・基準達成（50 lux 以下）：「良い光環境です」

---

### 3-3. プッシュ通知機能

睡眠リズムをサポートするため、ユーザーが設定した時刻にもとづいてローカル通知を送信する。

#### 通知の種類

| 通知タイミング | 内容 |
|---|---|
| 起床時刻 | 「朝の光を浴びる時間ですよ」 |
| 就寝時刻の45分前 | 「あと45分で就寝時間です。光を抑え、寝る準備をしましょう」 |

#### 設定仕様

- ユーザーは設定画面から起床時刻・就寝時刻を任意で設定できる
- 通知のON/OFFを設定画面から切り替え可能
- 通知はローカル通知として実装し、外部サーバーへの送信は行わない

---

### 3-4. シェア機能

計測中の画面をSNSなどに共有できる機能。

| 項目 | 仕様 |
|---|---|
| iOS | Share Sheet を使用 |
| Android | Intent 経由でシェア |
| 共有コンテンツ | スクリーンショット（現在の計測値・アドバイスを含む画面） |

---

### 3-5. フィードバック機能

ユーザーからの問い合わせ・要望を受け付ける窓口。

| 項目 | 仕様 |
|---|---|
| 実装方法 | メール送信（端末の標準メールアプリを起動） |
| アクセス場所 | 設定画面内 |

---

### 3-6. 広告

| 項目 | 仕様 |
|---|---|
| 広告ネットワーク | Google AdMob |
| 広告種別 | バナー広告のみ |
| 表示位置 | 画面下部 |
| インタースティシャル広告 | 非対応（実装しない） |

## 4. 非機能要件

### 4-1. デザイン

| 項目 | 仕様 |
|---|---|
| スタイル | シンプルミニマル |
| ダークモード | 非対応 |
| メインカラー | `#0e1a3a`（ネイビー）、`#5ba4cf`（スカイブルー） |
| サブカラー | メインカラーに近いトーンで調整（ベージュ・白系を想定） |

### 4-2. パフォーマンス

- アプリ起動後、即座に計測を開始できること
- リアルタイム更新においてUIがスムーズに動作すること

### 4-3. 保守性

- Flutter による単一コードベースで iOS・Android を管理
- 時間帯の境界値・アドバイステキストは定数として一元管理し、変更容易な設計とする

## 5. 画面構成

### 5-1. メイン画面（計測画面）

アプリ起動時に表示されるデフォルト画面。

| 要素 | 詳細 |
|---|---|
| lux 値表示 | 現在の照度を大きな数値でリアルタイム表示 |
| 時間帯ラベル | 「朝」「昼」「夜」など現在の時間帯を表示 |
| アドバイステキスト | 時間帯に応じた静的アドバイスを表示 |
| フィードバックテキスト | 測定値と基準値の比較結果を動的に表示 |
| 参考値注意表示 | 「この値はあくまで参考値です。正確な値ではありません。」を常時表示 |
| シェアボタン | 現在の画面をシェアするボタン |
| バナー広告 | 画面最下部に AdMob バナーを表示 |
| 設定画面への導線 | ナビゲーションバーまたはアイコンから設定画面へ遷移 |

### 5-2. 設定画面

| 要素 | 詳細 |
|---|---|
| 起床時刻設定 | 時刻ピッカーで起床時刻を設定 |
| 就寝時刻設定 | 時刻ピッカーで就寝時刻を設定 |
| 通知 ON/OFF | 通知全体のON/OFFをトグルで切り替え |
| フィードバック送信 | タップするとメールアプリを起動しお問い合わせメールを送信 |
| アプリバージョン | 現在のアプリバージョンを表示（例：Version 1.0.0） |

## 6. 技術スタック

| レイヤー | 採用技術 | 理由 |
|---|---|---|
| フレームワーク | Flutter | iOS・Android を単一コードベースで開発可能 |
| iOS 照度取得 | AVFoundation | カメラの ISO・シャッタースピードから lux を算出。外部依存なし |
| Android 照度取得 | SensorManager（TYPE_LIGHT） | OS標準の照度センサーAPIを直接利用 |
| プッシュ通知 | flutter_local_notifications | ローカル通知のクロスプラットフォーム対応 |
| 広告 | google_mobile_ads | AdMob の公式 Flutter プラグイン |
| シェア | share_plus | iOS Share Sheet・Android Intent に対応 |

## 7. 光量基準値と判定ロジック

### 基準値一覧

| 時間帯 | 時刻範囲 | 基準値 | アドバイス方針 |
|---|---|---|---|
| 朝 | 起床〜11:59 | 2,500 lux 以上 | 積極的に光を浴びることを促す |
| 昼 | 12:00〜17:59 | 目標 10,000 lux 以上 | 現状確認を促す |
| 夜 | 18:00〜就寝前 | 50 lux 以下 | 光を抑えることを促す |
| 測定上限 | — | 300,000 lux | これ以上は上限値として表示 |

### フィードバック判定フロー

```
現在時刻を取得
  ├─ 朝（起床〜11:59）
  │     ├─ 測定値 >= 2,500 lux → 「十分な光を浴びています」
  │     └─ 測定値 < 2,500 lux  → 「もう少し明るい場所へ移動しましょう」
  │
  ├─ 昼（12:00〜17:59）
  │     └─ 測定値を表示（「目標 10,000 lux 浴びましょう」）
  │
  └─ 夜（18:00〜就寝前）
        ├─ 測定値 <= 50 lux   → 「良い光環境です」
        └─ 測定値 > 50 lux    → 「光を抑えて寝る準備をしましょう」

測定値 >= 300,000 lux の場合 → 上限値（300,000 lux）として表示
```

## 8. 権限・プライバシー

### 必要な権限

| 権限 | プラットフォーム | 用途 |
|---|---|---|
| カメラ使用許可 | iOS のみ | 照度算出のためにカメラ映像を利用 |
| 通知送信許可 | iOS・Android | 起床・就寝リマインダーの送信 |

### プライバシー方針

| 項目 | 方針 |
|---|---|
| 個人データの外部送信 | 行わない |
| 測定値の保存 | ローカルデバイス内のみ（外部サーバーへの送信なし） |
| 広告SDK | Google AdMob が利用するデータはAdMobのプライバシーポリシーに準拠 |

## 9. 制約・注意事項

### 計測精度について

- iOSのカメラを使った照度算出は機種・カメラ性能・環境によって誤差が生じる
- Androidの照度センサーも機種によって精度が異なる
- アプリ内で「参考値であり正確な値ではない」旨を常時表示し、ユーザーの誤解を防ぐ

### 範囲外の動作

- 測定値が 300,000 lux を超えた場合は「300,000 lux」として上限表示する
- 0 lux 未満の値が返った場合は「0 lux」として下限表示する

### 対象外の機能（スコープ外）

- 測定値の履歴記録・グラフ表示
- インタースティシャル広告・リワード広告
- 有料プラン・サブスクリプション
- ダークモード対応
- 多言語対応（日本語のみ）
