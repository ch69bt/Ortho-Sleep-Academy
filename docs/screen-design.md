> アプリ名：Ortho Luxmeter

> プラットフォーム：iOS / Android（Flutter）
> 言語：日本語のみ
> Version 1.0

## 3. メイン画面（計測画面）

起動時のデフォルト画面。カメラ（iOS）または照度センサー（Android）を用いてリアルタイムに照度を計測し、時間帯に応じたアドバイスを表示する。

レイアウトは上から下の順に以下の要素で構成される。

---

### 3-1. ナビゲーションバー

| 要素 | 仕様 |
|---|---|
| タイトル | 「Ortho Luxmeter」を左寄せに表示。Primary色。 |
| 右アクション | 歯車アイコン（設定アイコン）。タップで設定画面へ遷移。 |
| 背景色 | Background（`#ffffff`） |
| ボトムシャドウ | なし |

---

### 3-2. 時間帯セクション

ナビゲーションバー直下に配置。現在時刻に基づいて時間帯を自動判定し、対応するアイコンとラベルを表示する。

| 時間帯 | アイコン | ラベル | 判定時刻の目安 |
|---|---|---|---|
| 朝 | ☀ 太陽 | 朝 | 設定した起床時刻 〜 12:00 |
| 昼 | ☁ 雲 | 昼 | 12:00 〜 18:00 |
| 夜 | 🌙 月 | 夜 | 18:00 〜 設定した起床時刻 |

- アイコン・テキストともにSecondary色（`#5ba4cf`）で表示
- テキストのletterSpacingは広めに設定（視認性向上）

---

### 3-3. Lux値表示セクション

画面中央の最大スペースを占める主役要素。

| 要素 | 仕様 |
|---|---|
| 数値 | 72sp、bold、Primary色。現在の照度値（整数）を表示。 |
| 単位「lux」 | 24sp、TextSecondary色。数値の右下に隣接して表示。 |
| 更新アニメーション | 数値変動時はスムーズなアニメーションで値が変化（急変動の場合も滑らかに推移）。 |

---

### 3-4. ゲージ・インジケーターセクション

計測値を視覚的に示す水平プログレスバー。

| 要素 | 仕様 |
|---|---|
| バー形状 | 水平プログレスバー（角丸）。全幅表示。 |
| スケール | 0 〜 300,000 lux の対数スケール |
| バー下ラベル | 左端：「0」、右端：「300,000」（TextSecondary色、小サイズ） |

#### バー色の条件分岐

| 条件 | バー色 |
|---|---|
| 時間帯：朝 かつ 測定値 ≥ 2,500 lux | Success（`#2ecc71`） |
| 時間帯：朝 かつ 測定値 < 2,500 lux | Warning（`#f39c12`） |
| 時間帯：夜 かつ 測定値 ≤ 50 lux | Success（`#2ecc71`） |
| 時間帯：夜 かつ 測定値 > 50 lux | Warning（`#f39c12`） |
| 時間帯：昼（常時） | Secondary（`#5ba4cf`） |

---

### 3-5. アドバイス・フィードバックセクション

Surface色（`#f4f8fc`）の角丸カード（borderRadius: 12）内に3要素を縦並びで表示。

#### アドバイステキスト（静的・時間帯別）

| 時間帯 | アドバイステキスト |
|---|---|
| 朝 | 「起床後は明るい光を浴びて体内時計をリセットしましょう」 |
| 昼 | 「日中は自然光を積極的に取り入れましょう」 |
| 夜 | 「就寝前は光を抑えてメラトニンの分泌を止めないようにしましょう」 |

- 16sp、TextPrimary色

#### フィードバックテキスト（動的・測定値ベース）

- 14sp、TextPrimary色
- 測定値と時間帯に応じてリアルタイムに変化するメッセージを表示
- 例：朝で基準達成時「十分な明るさです。このまま日光を浴び続けましょう」、朝で基準未達時「もう少し明るい場所へ移動するか、カーテンを開けてみましょう」

#### 基準値表示

- 12sp、TextSecondary色
- 朝：「基準値：2,500 lux 以上」
- 夜：「基準値：50 lux 以下」
- 昼：表示なし（または非表示）

---

### 3-6. 注意表示

カードセクション直下に固定テキストとして表示。

```
※この値はあくまで参考値です。正確な照度計ではありません。
```

- 11sp、TextSecondary色（`#6b7b99`）、中央寄せ

## 4. 設定画面

通知の有効/無効・起床就寝時刻・フィードバック送信・アプリバージョン確認が行える画面。セクションヘッダー付きのリスト形式で構成する。

---

### 4-1. ナビゲーションバー

| 要素 | 仕様 |
|---|---|
| タイトル | 「設定」（中央寄せ） |
| 左アクション | 戻るボタン（前の画面へ戻る） |
| 背景色 | Background（`#ffffff`） |

---

### 4-2. 通知設定セクション

**セクションヘッダー：「通知」**（12sp、TextSecondary色、大文字表記）

| 項目 | 種別 | 仕様 |
|---|---|---|
| 通知を受け取る | トグルスイッチ | ON/OFFで通知の有効・無効を切り替え。デフォルトOFF。 |
| 起床時刻 | ラベル + 時刻表示 | タップで時刻ピッカーを表示。通知OFFの場合はグレーアウト・タップ不可。 |
| 就寝時刻 | ラベル + 時刻表示 | タップで時刻ピッカーを表示。通知OFFの場合はグレーアウト・タップ不可。 |

- 通知がOFFの場合、起床時刻・就寝時刻のリスト行は視覚的にグレーアウトし、タップ操作を無効化する
- 時刻ピッカー：iOSは`CupertinoDatePicker`（時刻モード）、Androidは`TimePickerDialog`を使用

---

### 4-3. サポートセクション

**セクションヘッダー：「サポート」**（12sp、TextSecondary色、大文字表記）

| 項目 | 仕様 |
|---|---|
| フィードバックを送る | タップでメールアプリを起動。送信先：`support@example.com`、件名は自動設定（`[Ortho Luxmeter] フィードバック`）。 |

---

### 4-4. アプリ情報セクション

**セクションヘッダー：「アプリ情報」**（12sp、TextSecondary色、大文字表記）

| 項目 | 表示内容 |
|---|---|
| バージョン | ラベル「バージョン」+ 値「1.0.0」（右寄せ） |

## 5. 状態遷移・インタラクション

### 起動フロー

```
アプリ起動
  └→ スプラッシュなし
       └→ メイン画面（即座に計測開始）
```

### 画面遷移フロー

```
メイン画面
  └→ [歯車アイコンタップ]
       └→ 設定画面（モーダルまたはプッシュナビゲーション）
            └→ [戻るボタン]
                 └→ メイン画面
```

### 設定画面の通知トグル連動

```
通知トグル: OFF（デフォルト）
  └→ 起床時刻行：グレーアウト・タップ不可
  └→ 就寝時刻行：グレーアウト・タップ不可

通知トグル: ON
  └→ 起床時刻行：アクティブ・タップ可能
       └→ [タップ] → 時刻ピッカー表示
            └→ [決定] → 時刻を保存・表示更新・通知スケジュール更新
  └→ 就寝時刻行：アクティブ・タップ可能
       └→ [タップ] → 時刻ピッカー表示（iOS: CupertinoDatePicker / Android: TimePickerDialog）
            └→ [決定] → 時刻を保存・表示更新・通知スケジュール更新
```

### シェアフロー（メイン画面）

```
[シェアボタンタップ]
  └→ RepaintBoundaryでウィジェットのスクリーンショット取得
       └→ share_plusのShare.shareXFilesでシェアシート起動
            └→ テキスト：「現在の照度：{lux値} lux【Ortho Luxmeter】」
```
